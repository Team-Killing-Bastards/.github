## Operating Principles

### Operational value over feature volume

Projects should solve recurring gameplay or administration problems. Features are not added merely to make a project appear larger.

### Safety before automation

Bulk changes and dispatch automation require clear scope, preview or validation where practical, visible run state and reliable cancellation.

### Capability over labels

Vehicle names alone do not prove operational suitability. Training, personnel, equipment, patients and existing resource commitments must be considered.

### Traceable releases

Source versions, changes, migration risks and release artefacts should be documented and reproducible.

### Accurate attribution

Development credit must reflect the work actually performed.
